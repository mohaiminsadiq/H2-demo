Gentic Algorithm

In order to find bias in a ML model via burden on a specific class, we need to find “counterfactual” corresponding to each data point in the dataset.

Counterfactual is the point that is closest to the original point, but results in the opposite classification by the ML model.

Thus we can see the counterfactual as a measure, of how far is one individual from getting the opposite decision in a ML model, and burden is the average of this measure within some group of individuals (e.g. with a specific skin color, or a specific race).

We find counterfactuals using a genetic algorithm, which is demonstrated below. The genetic algorithm involves randomly generating points within sample space, randomly mutating some population, randomly crossoverring some features between some population, and filtering the candidates with better fitness (i.e. closer distance from the original individual) to find the optimal counterfactual.

Genetic Algorithm in Real Dataset

The above plot figure shows the evolution of the genetic algorithm in finding the counterfactual in the COMPAS dataset. Among many features, it is showing only 3 features to project in 3D space. The black point is the original individual, and the light blue point is the closest counterfactual. The red points are the ones that result in same classification, so cannot be the counterfactual. The blue points are the candidates with top 50% fitness (i.e. closeness), and the Green points are bottom 50% candidates for the counterfactual.

Since the ML model is much more complicated than the above case, the class bound boundary is harder to see compared to the above figure, but the red points (i.e. Non-counterfactuals) are somewhat separeted from candidates for the counterfactual that the algorithm is finding.

It can be seen that the counterfactual gradually moves toward the individual, along with the generations. In this case, the fitness reaches 0.46 at generation 10.
